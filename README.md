## 🧬 Who I Am

I'm an infrastructure automation engineer with deep roots in **Windows endpoint management** — SCCM/ConfigMgr, Intune, OSD task sequences, and enterprise-scale PowerShell tooling. I think in pipelines. I don't manually deploy anything that can be scripted, and I don't write scripts without thinking about how they'd live inside a CI/CD workflow.

My current trajectory: closing the gap between endpoint engineering and modern DevOps. That means Ansible, Terraform, Docker, and cloud-native infrastructure — applied to real systems, not just tutorials.

> 💡 *"If I'm doing it more than once, I'm automating it. If I'm automating it, I'm modularizing it. If I'm modularizing it, it belongs in a pipeline."*
